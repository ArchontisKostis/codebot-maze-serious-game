Overview > repeat
Control Structure Detail
repeat(count) … end

Repeats a block of commands exactly count times. The block may contain any number of move commands and other repeat blocks — nesting is fully supported.

Loop overhead costs no extra game frames. Only the move commands inside the block count toward your frame total.

Every repeat must be closed with a matching end. If the closing end is missing the program will not run and an error is shown with the line number.

Parameters:
count — integer between 1 and 100 (inclusive). Values outside this range are errors.
Examples
Repeat a single move

Move the robot right three times without writing moveRight three times.

repeat(3)
    moveRight
end
Repeat multiple moves

Every command inside the block is repeated. This moves the robot right then down, four times over. 

repeat(4)
    moveRight
    moveDown
end
Commands before and after a repeat

Mix standalone moves with a loop freely.

moveUp
repeat(3)
    moveRight
end
moveDown
Nested repeat blocks

A repeat block can contain another repeat block. Each end closes the innermost open repeat. The example below moves up once, then sweeps left 4 cells, and repeats that whole pattern twice. 

repeat(2)
    moveUp
    repeat(4)
        moveLeft
    end
    moveDown
end
Maximum count

The highest allowed count is 100.

repeat(100)
    moveRight
end
Valid Count Values
Value Valid? Reason
1 Yes Minimum allowed value
50 Yes Within range
100 Yes Maximum allowed value
0 No Must be at least 1
101 No Exceeds maximum of 100
